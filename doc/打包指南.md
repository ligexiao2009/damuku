# B站弹幕外挂 打包与使用指南

## 一、开发模式运行

```bash
# 1. 安装依赖
npm install

# 2. 配置 .env（复制 .env.example 修改）
cp .env.example .env
# 编辑 .env，填入 VIDEO_DIR、BILI_SESSDATA 等

# 3. 启动后端服务
node server.js

# 4. 另一个终端启动弹幕悬浮窗
npm run overlay
```

## 二、打包为 macOS DMG

```bash
# 使用国内镜像下载 Electron 二进制（解决超时问题）
ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/" npm run dist:mac
```

打包产物在 `dist/B站弹幕外挂-1.0.0-arm64.dmg`。

## 三、安装与配置

1. 双击 DMG 安装到 Applications
2. 首次启动会自动生成配置文件：
   ```
   ~/Library/Application Support/b-video-player/.env
   ```
3. 编辑该文件，填入你的视频目录和 B站 Cookie：
   ```
   VIDEO_DIR=/你的视频路径
   PORT=5001
   BILI_SESSDATA=你的SESSDATA
   DANMAKU_MAX_DURATION=10800
   DANMU_CACHE_DIR=/你的视频路径/danmu
   ```
4. 重启应用

> **注意：** 配置文件在应用升级时不会丢失。

## 五、常见问题

### Q1: npm install 报超时/网络错误

Electron 二进制包约 100MB，直连 GitHub 经常超时。使用国内镜像：

```bash
ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/" npm install
```

### Q2: 打包时下载 Electron 失败

同样用镜像：

```bash
ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/" npm run dist:mac
```

### Q3: 启动时闪退 / 弹出 EADDRINUSE 错误

端口被占用。杀掉旧进程：

```bash
lsof -ti:5001 | xargs kill
```

然后到 `.env` 换个端口也可。

### Q4: 弹幕外挂窗口不显示

依次检查：
1. 是否启动了服务器（`node server.js`）
2. 浏览器打开 `http://localhost:5001/overlay.html` 确认页面可访问
3. 按 `Ctrl+O` 看控制面板是否弹出
4. 是否给了辅助功能权限（macOS 系统设置 → 隐私与安全性 → 辅助功能）

### Q5: 控制面板弹出但弹幕不显示

1. 确认已输入 BV 号并点击"加载弹幕"
2. 点击"开始"启动弹幕引擎
3. 检查 BILI_SESSDATA 是否配置（登录 B站 后按 F12 → Application → Cookies → SESSDATA）
4. 检查配置文件端口与实际运行端口一致

### Q6: 输入法候选窗不显示 / 被遮挡

这是 Electron 透明窗口的已知限制。变通方案：
- 把控制面板拖到屏幕上方，输入法候选窗会在下方展开
- 或先在别处打几个字再回来操作

### Q7: 弹幕时间与视频不同步

1. 点"显示时间戳"查看每条弹幕的原始时间
2. 调整"偏移"滑块同步
3. 在进度条拖动到大致位置，用时间输入框精确跳转（支持 `1:23:45` 格式）

### Q8: 外接显示器弹幕窗口大小异常

每个显示器各有一个独立弹幕窗口，分辨率不同面板大小可能略有差异，不影响弹幕渲染。

### Q9: 配置文件不生效

开发模式读项目目录的 `.env`，打包版读 `~/Library/Application Support/b-video-player/.env`。确认编辑的是对应文件。

### Q10: 如何更新 BILI_SESSDATA

打开配置文件修改后重启应用即可。B站 SESSDATA 有效期约 3-7 天，过期后需要重新获取。

## 六、版本更新方式

重新打包 DMG，覆盖安装即可。配置文件独立存储，不会丢失。

## 版本号问题
每次 npm run dist:mac 会自动把 1.0.0 → 1.0.1 → 1.0.2 ……

生成的 DMG 文件名也会自动带新版本号。如果想手动跳到 2.0.0，跑一次 npm version minor --no-git-tag-version
