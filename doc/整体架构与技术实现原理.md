# B站弹幕外挂 — 整体架构与技术实现原理

## 一、项目概述

一套跨平台弹幕悬浮系统，支持 B站、腾讯视频、芒果TV、直播吧、腾讯体育五种弹幕源，覆盖桌面 Electron 透明悬浮窗、Web 播放器、iPad 直播流播放三种使用场景。集成 IINA 播放器联动，实现播放进度、暂停/播放、快进/快退全自动同步。

### 核心能力

- 本地视频播放 + 在线弹幕叠加
- 多弹幕源切换（VOD 一次性加载 + 2 种直播实时轮询）
- Electron 透明置顶窗口，叠加于任意播放器之上
- iPad 触屏优化，支持 HLS 直播流 + 实时弹幕
- IINA 播放器深度联动：进度同步、暂停同步、快进跟随、自动切换视频
- 视频转码（ffmpeg videotoolbox 硬加速）
- 播放进度记忆、弹幕缓存备份、批量重命名
- PWA 主屏幕图标支持

## 三、弹幕引擎：DanmakuEngine

### 3.1 设计理念

`DanmakuEngine` 是一个纯 JS 类，不依赖任何框架，只需要一个 DOM 容器即可工作。桌面端和 iPad 端共享同一份代码。

### 3.2 渲染架构

**轨道系统 (Lane System)**

将可视区域纵向划分为多个轨道（`fontSize + 8px` 高度），每条弹幕分配到最早可用的轨道，通过 `_laneAvailableAt[]` 时间戳数组避免弹幕重叠。

**DOM 对象池 (Object Pool)**

弹幕 DOM 元素复用而非创建/销毁。滑出屏幕的元素在回收前立即移出屏幕（`translate3d(-9999px, -9999px, 0)`），避免复用瞬间闪烁。

**手动动画循环 (Manual rAF)**

使用 `requestAnimationFrame` 手动计算每帧的 `translate3d` 变换，这是 Electron 透明窗口下的最佳实践——CSS/WAF 在透明窗口中不可靠。

### 3.3 公开 API

| 方法 | 用途 |
|------|------|
| `configure(opts)` | 批量设置 speed/area/offset/fontSize/opacity |
| `load(danmus)` | 一次性替换全部弹幕（VOD 场景） |
| `append(danmus)` | 追加新弹幕（直播轮询场景） |
| `seek(time)` | 跳转到指定时间，重置弹幕索引 |
| `start(time)` / `pause()` / `resume(time)` | 播放生命周期 |
| `destroy()` | 清理：取消 rAF、清空 DOM、断开 ResizeObserver |

### 3.4 弹幕数据格式

```js
{
  text:  "弹幕内容",
  time:  34.567,     // 视频时间（秒）
  color: "#ffffff",  // 颜色
  mode:  1,          // 1=滚动 4=底部 5=顶部
  ctime: 1712345678  // Unix 时间戳（可选，用于时间戳显示）
}
```

## 四、弹幕源服务

### 4.1 B站 (bilibili.js) — VOD

**两种策略：**

| 策略 | 接口 | 格式 | 特点 |
|------|------|------|------|
| XML | `comment.bilibili.com/{cid}.xml` | XML | 老接口，稳定，弹幕较少 |
| seg.so | `api.bilibili.com/x/v2/dm/web/seg.so` | Protobuf | 新接口，弹幕更多，默认策略 |

**seg.so 分段加载流程：**

```
BV/EP → fetchVideoMeta() → cid + duration
  → segmentCount = ceil(duration / 360)，上限 100
  → 依次 fetchDanmuSeg(cid, aid, segIdx)，间隔 2s
  → tryParseDanmuSeg() 解码 protobuf
  → 失败分段 5s 后重试一次
  → danmuProgressMap 跟踪进度，前端轮询显示
```

**认证：** `BILI_SESSDATA` 环境变量，自动拼为 `SESSDATA=xxx` Cookie。

### 4.2 腾讯视频 (tencent.js) — VOD

- 接口：`dm.video.qq.com/barrage/segment/{vid}/t/v1/{start}/{end}`
- 每 30 秒一段，毫秒单位
- 解析 `barrage_list`，支持 `content_style` 渐变颜色
- 单段失败靜默跳过

### 4.3 芒果TV (mango.js) — VOD

- 接口：`bullet-ws.hitv.com/bullet/tx/{date}/{time}/{videoId}/{index}.json`
- 序号递增探测，连续两次 `NoSuchKey` 后停止

### 4.4 直播吧 (zhibo8.js) — 实时轮询

**去重机制：前端维护 `lastMaxId`，传给后端过滤。多客户端独立，互不干扰。**

```
前端 lastMaxId=0 → GET /api/danmaku?source=zhibo8&lastMaxId=0
  → fetchCount() → fetchPage(latest) → 筛选 id > lastMaxId
  → 返回 {danmus, maxId: 635}
前端 存 lastMaxId=635
下次请求 lastMaxId=635 → 只返回 id > 635 的新弹幕
```

**关键设计：**
- 服务端无状态，前端自己维护 `lastMaxId`
- `time=0` 由前端填充当前播放时间
- URL 中 `type` 参数支持 `zuqiu`/`nba`/`other`
- `matchId` 原样传递，不做数字提取
- 按 `room` 字段区分颜色：球迷(1)=白色，彩民(2)=金色

### 4.5 腾讯体育 (txsp.js) — 实时轮询

**与直播吧的区别：** POST 请求、cursor 分页、服务端控制轮询间隔。

```
POST zbpbaccess.video.qq.com/...LiveDanmuRead/PullMsgH5
body: {room_id, program_id, cookie: cursor}
headers: {Cookie: <用户登录态>}

响应：
  cookie (next cursor) → 分页游标
  pull_interval → 服务端建议轮询间隔
  nonconst_speed_msg_info[] → 弹幕 (msg_id=33)
  const_speed_msg_info[] → 点赞 (msg_id=48)

去重：前端维护 lastSeq，后端按 msg_seq 过滤
```

**配置方式：** 不需要 `.env`。前端粘贴 curl/fetch → 自动提取 room_id + program_id + cookie，存在前端内存里。

## 五、统一 API：/api/danmaku

### 接口设计

```
GET /api/danmaku?source={bili|qq|mango|zhibo8|txsp}&id={videoId}&duration={ms}&type={matchType}
```

| source | id 格式 | 其他参数 | 缓存 |
|--------|---------|----------|------|
| bili | BV号 / EP号 | strategy=xml\|seg.so | 是 |
| qq | VID | duration(ms) | 是 |
| mango | HHMMSS/videoId | — | 是 |
| zhibo8 | 比赛ID（原样） | type=zuqiu\|nba\|other, lastMaxId | 否 |
| txsp | 不需要 id | roomId, programId, lastSeq, cursor | 否 |

### 缓存策略

- VOD 源：缓存到 `cache/danmu/` 目录，`refresh=1` 强制刷新。刷新前自动备份旧文件到 `.bak.YYYYMMDDHHmm`，精准到分钟。
- 直播源（zhibo8/txsp）：不缓存，每次请求实时拉取。

|--------|------|
| `Cmd/Ctrl + Space` | 播放 / 暂停弹幕 |
| `Cmd/Ctrl + ←/→` | 快退/快进 0.5s |
| `Cmd/Ctrl + Shift + ←/→` | 快退/快进 2s |
| `Cmd/Ctrl + ↑/↓` | 透明度 ±5% |
| `Cmd/Ctrl + Shift + ↑/↓` | 字号 ±2px |
| `Cmd/Ctrl + O` / `Esc` | 显示/隐藏控制面板 |

### 6.3 快捷键流程

```
main.js globalShortcut.register()
  → 按键触发 → win.webContents.send('shortcut:xxx')
  → preload.js ipcRenderer.on('shortcut:xxx')
  → window.electronAPI.onShortcut()
  → overlay.js 回调处理
```

### 6.4 窗口穿透机制

- 弹幕层：`pointer-events: none` — 鼠标穿透到下方播放器
- 面板显示时：`setClickThrough(false)` — 面板可点击
- 面板隐藏时：`setClickThrough(true)` — 恢复穿透

### 6.5 全屏 Space 保持

`startVisibilityHeartbeat()` 每 3 秒重新执行 `setAlwaysOnTop` + `setVisibleOnAllWorkspaces`，防止 macOS 全屏 Space 切换后窗口丢失。

## 七、IINA 播放器联动

### 7.1 原理

IINA 是基于 mpv 的播放器，支持 JSON IPC。配置 `input-ipc-server=/tmp/iina-socket` 后，外部程序通过 Unix Domain Socket 直接通信。

### 7.2 架构

```
IINA (mpv) ←── Unix Socket ──→ iina-watcher.py
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
              POST /api/     POST /api/     GET /api/progress
              progress      iina-state     (恢复进度)
                    │             │
                    ▼             ▼
               server.js     server.js
                    │             │
                    ▼             ▼
              playback/     iinaState
              (进度文件)    {path, time, paused}
                                 │
                    overlay.js 轮询 (2s)
                         │
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
     自动切换文件夹   暂停/播放同步   快进/快退跟踪
     自动选视频       (pause sync)   (gap>30s → seek)
     自动加载弹幕
```

### 7.3 自动联动功能

| IINA 动作 | overlay 响应 |
|-----------|-------------|
| 打开视频 | 自动切换文件夹 + 选中视频 + 识别弹幕源 + 恢复进度 + 加载弹幕 |
| 暂停 | 弹幕暂停 |
| 播放 | 弹幕恢复 |
| 快进/快退 (>30s) | 弹幕 seek 跟随 |
| 退出 | 弹幕进度自动保存 |

### 7.4 弹幕源优先级

文件名检测到腾讯 VID 和 B站 BV 同时存在时，优先选腾讯视频弹幕源。`detectVid` 自动排除 `BV` 开头的字符串避免误判。

## 八、iPad 端 (ipad.html + ipad.js)

### 8.1 独立 JS 文件

iPad 端不复用 overlay.js，完全独立。原因：
- overlay.js 的 `loadFolders()`/`folder-history` 会恢复上次文件夹选择，与 iPad 直播场景冲突
- iPad 锁定直播吧/腾讯体育等实时源，不需要本地文件浏览
- 独立文件避免事件冲突、异步时序问题

### 8.2 特殊功能

- **默认直播吧**：sourceSelect 默认选中 `zhibo8`
- **红点触控**：右侧垂直居中，18×18px 可点击，隐藏时脉冲动画提醒
- **自动隐藏**：面板无操作 8 秒自动隐藏
- **📋 两段式粘贴**：第一次清空输入框，第二次聚焦+全选（引导手动粘贴，绕过 iOS Safari HTTP 下 Clipboard API 限制）
- **localStorage 持久化**：直播地址 + 比赛 ID 刷新不丢失
- **PWA 主屏幕**：`apple-mobile-web-app-capable` 全屏启动，`apple-touch-icon` 粉色图标
- **直播流错误处理**：三层兜底（fetch HEAD 预检 / HLS.js 致命错误 / video error 事件）

### 8.3 与桌面端差异

| 特性 | 桌面端 (overlay.js) | iPad 端 (ipad.js) |
|------|-------------------|-------------------|
| JS 文件 | overlay.js | ipad.js（独立） |
| 默认弹幕源 | B站 | 直播吧 |
| 视频播放 | 外部播放器 | 内嵌 `<video>` + HLS.js |
| 面板控制 | 键盘快捷键 + ✕ 按钮 | 红点触摸 + ✕ 按钮 + 8s 自动隐藏 |
| 启动行为 | 显示面板 3s 后隐藏 | 显示面板 8s 后隐藏 |
| 进度/跳转 | 显示 | 隐藏 |
| 文件夹选择 | 显示（直播源时隐藏） | 隐藏 |
| 持久化 | 服务端 + localStorage | localStorage |
| 粘贴功能 | txsp 智能提取 | 直播流两段式粘贴 + txsp 智能提取 |

## 十一、数据流总结

### VOD 弹幕加载（B站/腾讯/芒果）

```
用户输入 ID → /api/danmaku?source=bili&id=xxx
  → fetchVideoMeta() → cid + duration
  → 分段抓取 → 解析 → 缓存到硬盘（刷新前备份旧文件）
  → 返回 JSON → DanmakuEngine.load() → rAF 按时序渲染
```

### 实时弹幕轮询（直播吧/腾讯体育）

```
用户输入 ID → 前端 setInterval/setTimeout 轮询
  → /api/danmaku?source=zhibo8&id=xxx&lastMaxId=n
  → 服务端无状态，前端维护 lastMaxId/lastSeq
  → 返回增量弹幕 → DanmakuEngine.append() → 立即渲染
```

### IINA 全自动联动

```
IINA 打开视频 → watcher POST path → /api/iina-state
  → overlay 轮询检测 → 切换文件夹 → 选视频
  → 识别弹幕源 → 恢复进度 → 自动加载弹幕

IINA 播放中 → watcher POST time → overlay 进度同步（去重）
IINA 暂停   → watcher POST paused → overlay 暂停弹幕
IINA 快进   → gap > 8s → overlay seek 跟随
```
