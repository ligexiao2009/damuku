# B站弹幕外挂 — 整体架构与技术实现原理

## 一、项目概述

一套跨平台弹幕悬浮系统，支持 B站、腾讯视频、芒果TV、直播吧四种弹幕源，覆盖桌面 Electron 透明悬浮窗、Web 播放器、iPad 直播流播放三种使用场景。

### 核心能力

- 本地视频播放 + 在线弹幕叠加
- 多弹幕源切换（VOD 一次性加载 + 直播实时轮询）
- Electron 透明置顶窗口，叠加于任意播放器之上
- iPad 触屏优化，支持 HLS 直播流 + 实时弹幕
- 视频转码（ffmpeg videotoolbox 硬加速）
- 播放进度记忆、弹幕缓存、批量重命名

## 三、弹幕引擎：DanmakuEngine

### 3.1 设计理念

`DanmakuEngine` 是一个纯 JS 类，不依赖任何框架，只需要一个 DOM 容器即可工作。桌面端和 iPad 端共享同一份代码。

### 3.2 渲染架构

**轨道系统 (Lane System)**

将可视区域纵向划分为多个轨道（`fontSize + 8px` 高度），每条弹幕分配到最早可用的轨道，通过 `_laneAvailableAt[]` 时间戳数组避免弹幕重叠。

```
屏幕宽度
├─────────────────────────────────────┤
│ 轨道0: 弹幕A ──────────────►         │  _laneAvailableAt[0] = now + 间距
│ 轨道1:    弹幕B ───────────►         │  _laneAvailableAt[1] = now + 间距
│ 轨道2:        弹幕C ───────►         │
└─────────────────────────────────────┘
```

**DOM 对象池 (Object Pool)**

弹幕 DOM 元素复用而非创建/销毁。滑出屏幕的元素移入 `_pool` 数组，新弹幕从中取出复用，减少 GC 压力和重排。

**手动动画循环 (Manual rAF)**

使用 `requestAnimationFrame` 手动计算每帧的 `translate3d` 变换，而非依赖 CSS Animations 或 Web Animations API。这是 Electron 透明窗口下的最佳实践——后者在透明窗口中不可靠，弹幕会卡顿或消失。

```
每帧执行:
  1. _tickActive(now)  → 更新所有活跃弹幕的 translate3d 位置
  2. 计算插值时间      → currentTime + (now - lastPerfTime) / 1000
  3. _flush(time)      → 发射到达时间点的弹幕（最多 6 条/帧）
  4. requestAnimationFrame(tick)
```

**文字宽度测量**

使用离屏 `<canvas>` + `measureText()` 测量弹幕文字宽度，结果缓存于 `_widthCache` Map，key 为 `fontSize:text`。

### 3.3 公开 API

| 方法 | 用途 |
|------|------|
| `configure(opts)` | 批量设置 speed/area/offset/fontSize/opacity |
| `load(danmus)` | 一次性替换全部弹幕（VOD 场景） |
| `append(danmus)` | 追加新弹幕（直播轮询场景） |
| `seek(time)` | 跳转到指定时间，重置弹幕索引 |
| `start(time)` / `pause()` / `resume(time)` | 播放生命周期 |
| `destroy()` | 清理：取消 rAF、清空 DOM、断开 ResizeObserver |

### 3.4 弹幕数据格式

```js
{
  text:  "弹幕内容",
  time:  34.567,     // 视频时间（秒）
  color: "#ffffff",  // 颜色
  mode:  1,          // 1=滚动 4=底部 5=顶部
  ctime: 1712345678  // Unix 时间戳（可选，用于时间戳显示）
}
```

## 四、弹幕源服务

### 4.1 B站 (bilibili.js)

**两种策略：**

| 策略 | 接口 | 格式 | 特点 |
|------|------|------|------|
| XML | `comment.bilibili.com/{cid}.xml` | XML | 老接口，稳定，弹幕较少 |
| seg.so | `api.bilibili.com/x/v2/dm/web/seg.so` | Protobuf | 新接口，弹幕更多，默认策略 |

**seg.so 分段加载流程：**

```
BV/EP → fetchVideoMeta() → cid + duration
  → segmentCount = ceil(duration / 360)，上限 100
  → 依次 fetchDanmuSeg(cid, aid, segIdx)，间隔 2s
  → tryParseDanmuSeg() 解码 protobuf
  → 失败分段 5s 后重试一次
  → danmuProgressMap 跟踪进度，前端轮询显示
```

**认证：** 可选 `BILI_SESSDATA` Cookie 提高番剧弹幕获取成功率。

### 4.2 腾讯视频 (tencent.js)

- 接口：`dm.video.qq.com/barrage/segment/{vid}/t/v1/{start}/{end}`
- 每 30 秒一段，毫秒单位
- 解析 `barrage_list`，支持 `content_style` 渐变颜色
- 单段失败静默跳过

### 4.3 芒果TV (mango.js)

- 接口：`bullet-ws.hitv.com/bullet/tx/{date}/{time}/{videoId}/{index}.json`
- 序号递增探测，连续两次 `NoSuchKey` 后停止
- ID 格式：`HHMMSS/videoId`

### 4.4 直播吧 (zhibo8.js) — 唯一实时轮询源

与 VOD 源不同，直播吧是**增量拉取**：

```
┌──────────────────────────────────────────┐
│  状态机 (按 type+matchId 分区)            │
│  { lastNum: 0, lastMaxId: 0, perPage: 5 }│
└──────────────────────────────────────────┘
          │
          ▼
  fetchCount() → GET match{v}_count.htm
          │        返回 {num: "637", per_page: 5}
          ▼
  num === lastNum ? → 无新弹幕，return []
  num !== lastNum ? → page = ceil(num/perPage) - 1
          │
          ▼
  fetchPage(page) → GET match{v}_{page}.htm
          │        返回 [{id, createtime, content}, ...]
          ▼
  筛选 id > lastMaxId → 返回增量弹幕 [{text, time:0, rawTime, ...}]
```

**关键设计：**
- 服务端维护轮询状态，前端每次请求只获取增量
- `time=0` 由前端填充当前播放时间，确保弹幕立即出现
- URL 中 `type` 参数支持 `zuqiu`/`nba`/`other`

## 五、统一 API：/api/danmaku

### 接口设计

```
GET /api/danmaku?source={bili|qq|mango|zhibo8}&id={videoId}&duration={ms}&type={matchType}
```

| source | id 格式 | 其他参数 | 缓存 |
|--------|---------|----------|------|
| bili | BV号 / EP号 | strategy=xml\|seg.so | 是 |
| qq | VID | duration(ms) | 是 |
| mango | HHMMSS/videoId | — | 是 |
| zhibo8 | 比赛ID（自动提取数字） | type=zuqiu\|nba\|other | 否 |

### 统一响应格式

```json
{
  "code": 0,
  "data": {
    "source": "zhibo8",
    "id": "1975991",
    "count": 5,
    "danmus": [
      { "text": "好球！", "time": 0, "color": "#ffffff", "mode": 1, "rawTime": "2024-06-15 20:30:45" }
    ]
  }
}
```

### 缓存策略

- VOD 源（bili/qq/mango）：缓存到 `cache/danmu/` 目录，`refresh=1` 强制刷新
- 直播源（zhibo8）：不缓存，每次请求实时拉取

## 七、iPad 端 (ipad.html + ipad.js)

### 7.1 与桌面端的差异

| 特性 | 桌面端 (overlay.js) | iPad 端 (ipad.js) |
|------|-------------------|-------------------|
| JS 文件 | 共用 overlay.js | 独立 ipad.js，不复用 |
| 默认弹幕源 | B站 | 直播吧 |
| 视频播放 | 外部播放器 | 内嵌 `<video>` + HLS.js |
| 面板控制 | 键盘快捷键 | 红点触摸 + ✕ 按钮 + 8s 自动隐藏 |
| 进度/跳转 | 显示 | 隐藏 |
| 文件夹选择 | 显示 | 隐藏 |
| 持久化 | 服务端 `/api/overlay-config` | localStorage（直播地址 + 比赛ID） |
| 粘贴功能 | 无 | 📋 一键粘贴直播地址 |
| 快捷键 | 全量组合键 | 无（纯触摸） |

### 7.2 独立 JS 的原因

iPad 端不复用 overlay.js，因为两者的核心逻辑冲突：
- overlay.js 包含 `loadFolders()`/`folder-history` 等桌面端逻辑，会自动恢复上次播放的视频文件并切换弹幕源
- iPad 需要锁定直播吧为默认源，且不需要任何本地文件浏览功能
- 独立文件避免了事件冲突、异步时序问题和多余的 DOM 依赖

### 7.3 直播流错误处理

三层错误兜底：

```
1. fetch HEAD 预检 → HTTP 非200 → 显示状态码
2. HLS.js 致命错误 → 显示错误类型 + HTTP 状态
3. <video> error 事件 → 显示 MEDIA_ERR_* 代码
```

## 八、视频处理

### 8.1 直播流方案

支持 MP4 + H.264 + AAC 格式直接流式输出，其他格式通过 ffmpeg 实时转码。

### 8.2 转码

- 编码器：`h264_videotoolbox`（macOS 硬件加速）
- 音频：AAC 192k
- 输出：`frag_keyframe+empty_moov` 分片 MP4（支持流式播放）
- 传输：`Transfer-Encoding: chunked`，客户端断开自动 kill ffmpeg 进程

## 十、数据流总结

### VOD 弹幕加载（B站/腾讯/芒果）

```
用户输入 ID → 前端 /api/danmaku → server.js 路由
  → 对应 service 抓取 → 解析 → 缓存到磁盘
  → 返回 JSON → DanmakuEngine.load() → rAF 按时序渲染
```

### 实时弹幕轮询（直播吧）

```
用户输入 matchId → 前端 setInterval 2s
  → /api/danmaku?source=zhibo8&id=xxx&type=zuqiu
  → zhibo8.js fetchCount() → num 变化检测
  → fetchPage() → 筛选 id > lastMaxId
  → 返回增量弹幕 → DanmakuEngine.append() → 立即渲染
```

### 直播流播放（iPad）

```
用户输入/粘贴 M3U8 → fetch HEAD 预检
  → HTTP 200 → Safari 原生播放 / HLS.js 播放
  → HTTP 非200 → 状态栏显示错误
  → 播放错误 → video error 事件 → 状态栏显示错误
```

## 十一、使用方法

### 11.1 环境要求

- Node.js 18+
- macOS / Windows
- 可选：FFmpeg（用于视频转码）

### 11.2 安装

```bash
cd b-video
npm install
```

### 11.3 配置 .env

```env
# 视频文件目录
VIDEO_DIR=/Users/yourname/video

# 弹幕缓存目录
DANMU_CACHE_DIR=/Users/yourname/video/danmu

# 服务端口
PORT=5001

# B站 SESSDATA（可选，提高番剧弹幕获取成功率）
BILI_SESSDATA=your_sessdata_here
```

### 11.4 启动方式

#### 桌面悬浮窗（Electron）

```bash
# 一键启动（服务 + Electron 窗口）
./scripts/start_overlay.sh

# 或分步启动：
npm start                    # 终端1：启动后端服务
npm run overlay              # 终端2：启动 Electron 悬浮窗
```

启动后，悬浮窗自动覆盖在所有显示器上。在任意视频播放器（浏览器、IINA、PotPlayer 等）播放视频，输入 BV号/EP号/VID/比赛ID，加载弹幕即可叠加显示。

**控制面板操作：**
- `Cmd/Ctrl + O` 或 `Esc` — 打开/关闭控制面板
- 面板支持拖拽移动，双击标题栏归位
- 输入视频 ID → 点击"加载弹幕"
- 弹幕设置：偏移、字号、透明度、速度、显示区域

**键盘控制（弹幕时间轴）：**
- `Cmd/Ctrl + Space` — 播放/暂停弹幕
- `Cmd/Ctrl + ←/→` — 快退/快进 0.5 秒
- `Cmd/Ctrl + Shift + ←/→` — 快退/快进 2 秒
- `Cmd/Ctrl + ↑/↓` — 透明度调整
- `Cmd/Ctrl + Shift + ↑/↓` — 字号调整

#### iPad 直播页

```bash
npm start
```

然后在 iPad Safari 访问 `http://<Mac局域网IP>:5001/ipad.html`

**操作流程：**
1. 粘贴或输入 M3U8 直播流地址 → 点击 📋 一键粘贴剪贴板
2. 点击"播放直播"
3. 输入比赛 ID（如 `1975991`，支持粘贴完整 URL 自动提取数字）
4. 选择比赛类型（足球 / NBA / 其他）
5. 点击"加载弹幕" — 弹幕实时轮询，每条弹幕到达即显示
6. 点击"全屏"进入全屏模式

**触摸控制：**
- 右侧红点 — 点击唤出/隐藏控制面板
- 面板 ✕ 按钮 — 关闭面板
- 面板无操作 8 秒 — 自动隐藏
- 弹幕设置 — 点击"弹幕设置"展开

直播地址和比赛 ID 自动保存到浏览器本地存储，刷新页面不丢失。

#### Web 播放器（video.html）

```bash
npm start
```

浏览器访问 `http://localhost:5001/video.html`

**功能：**
- 选择本地视频文件或文件夹
- 自动匹配 B站 BV号 / EP号 / 腾讯 VID
- 播放列表管理（上一集/下一集）
- 播放进度自动记忆
- 全屏播放

### 11.5 弹幕源使用说明

| 弹幕源 | 输入格式 | 示例 |
|--------|---------|------|
| B站 | BV号 或 EP号 | `BV1CqizYmEWG` / `ep473307` |
| 腾讯 | VID | `z4102ir7m5b` |
| 芒果 | HHMMSS/videoId | `203000/123456` |
| 直播吧 | 比赛ID（数字） | `1975991`（支持粘贴完整URL自动提取） |

**B站弹幕策略选择：**
- `seg.so`（默认）：新接口，弹幕更完整，推荐
- `xml`：老接口，弹幕较少但稳定

### 11.6 打包发布

```bash
npm run dist:mac    # macOS DMG
npm run dist:win    # Windows NSIS 安装包
```

打包后的 app 首次启动会自动在 `~/Library/Application Support/B站弹幕外挂/` 创建 `.env` 模板，编辑该文件配置视频目录后重启生效。

### 11.7 视频转码

对于不支持浏览器播放的格式（如 MKV），可以通过 API 转码：

```bash
# 查看可用视频
curl http://localhost:5001/api/videos

# 发起转码
curl -X POST http://localhost:5001/api/convert \
  -H "Content-Type: application/json" \
  -d '{"name": "my_video.mkv"}'

# 查看转码状态
curl http://localhost:5001/api/convert/status?id=<taskId>
```

也可以在 `video.html` 页面中点击视频旁的转换按钮。

转码使用 macOS `h264_videotoolbox` 硬件加速，输出 MP4 + AAC 格式。

### 11.8 批量重命名

通过 `rename.html` 页面，输入文件夹路径和 B站番剧链接，自动将视频文件重命名为 `EP{XX}_{标题}_ep{id}.mp4` 格式，方便弹幕自动匹配。
